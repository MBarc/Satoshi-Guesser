For scale

  • 6.87 trillion keys/day — about 34× the number of stars in the Milky Way, every day.
  • The Milky Way has ~200 billion stars. We check that many keys every 2.4 seconds.
  • The universe is ~13.8 billion years old. Expected time to find a match is ~1047 times that.
  • Even at maximum theoretical optimization — full GPU farm, every CPU on Earth — finding a key is closer to confirming Bitcoin works than threatening it.

How it works

Each spin draws a cryptographically random 256-bit number and treats it as a Bitcoin private key. From that, secp256k1 elliptic-curve math derives a public key, then SHA-256 and RIPEMD-160 produce a 20-byte hash160. That hash gets compared against a local list of ~22,000 Patoshi-pattern addresses — the wallets widely attributed to Satoshi.

It runs on GitHub Actions across 20 parallel workers, each running 4 native-Rust threads. Each thread checks roughly 1 million keys per second. The elliptic-curve math goes through libsecp256k1's hand-tuned field arithmetic via FFI — scalar multiplication, then a Jacobian walk that accumulates N points via cheap mixed additions, then a Montgomery-trick pass that batch-inverts all N denominators with a single field inversion. The hashing stage stays on the Rust side: sha2 with SHA-NI hardware acceleration for SHA-256, then a hand-rolled 8-way AVX2 RIPEMD-160 that processes 8 candidates per SIMD instruction. No server, no cloud bill, no external API calls — just math.
